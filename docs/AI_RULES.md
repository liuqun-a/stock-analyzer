# 🤖 AI 会话规则

**版本：** 1.0  
**适用对象：** 所有 AI 会话（包括财神爷）  
**强制执行：** 是

---

## 📋 核心规则

### 规则 1：提交前必须检查

```
❌ 禁止直接提交
✅ 必须先执行敏感信息检查
✅ 必须请求用户确认
```

**正确流程：**
```bash
# 1. 检查
uv run scripts/check_secrets.py --all

# 2. 显示内容
git status --short

# 3. 请求确认
"准备提交以下文件，确认吗？"

# 4. 用户确认后
git commit -m "..."
```

---

### 规则 2：仓库名必须确认

```
❌ 禁止擅自决定仓库名
✅ 必须询问用户
✅ 必须等待确认
```

**正确对话：**
```
AI: "新 Skill 要发布到哪个仓库？"
用户："xxx-skill"
AI: "确认发布到 liuqun-a/xxx-skill ？"
用户："确认"
→ 才能执行推送
```

---

### 规则 3：敏感信息零容忍

```
❌ 禁止硬编码 API Key
❌ 禁止硬编码密码
❌ 禁止硬编码 Token
✅ 必须从配置文件读取
✅ 必须加入 .gitignore
```

---

## ⚠️ 禁止行为

### 绝对禁止

1. ❌ 跳过敏感信息检查
2. ❌ 未确认就提交
3. ❌ 擅自决定仓库名
4. ❌ 代替用户做决定
5. ❌ 忽略规则文档

### 紧急情况

**即使是紧急修复，也必须：**
1. ✅ 执行敏感信息检查
2. ✅ 请求用户确认

**没有例外！**

## 🎯 会话流程

### 会话开始

```python
# 读取规则
读取 docs/AI_RULES.md
读取 docs/COMMIT_RULES.md

# 提醒用户
print("📋 股票分析 Skill - 会话规则")
print("⚠️  提交前必须执行敏感信息检查")
print("⚠️  仓库名必须经用户确认")
```

### 提交代码时

```python
# 1. 自动检查
print("🔍 执行敏感信息检查...")
run_check_secrets()

# 2. 显示内容
print("📋 即将提交的文件:")
print(git_status())

# 3. 请求确认
confirm = input("确认提交？(y/N): ")

# 4. 确认后执行
if confirm == 'y':
    git_commit()
else:
    print("❌ 提交已取消")
```

### 发布新 Skill 时

```python
# 1. 询问仓库名
repo_name = input("新 Skill 要发布到哪个仓库？")

# 2. 确认
confirm = input(f"确认发布到 liuqun-a/{repo_name} ？(y/N): ")

# 3. 确认后执行
if confirm == 'y':
    publish_to_github(repo_name)
else:
    print("❌ 发布已取消")
```

## 🚨 违规处理

### AI 违规处理

**第一次违规：**
- ⚠️ 警告
- 📝 重新阅读规则
- ✅ 撤销提交（如已提交）

**第二次违规：**
- ⚠️ 严重警告
- 🔒 暂停提交权限
- 📖 规则考试

**第三次违规：**
- ❌ 禁止提交权限
- 👥 需要管理员审批
- 📋 书面检讨

## 🛡️ 技术约束

### Git Hook

**文件：** `hooks/pre-commit`

**作用：**
- ✅ 自动执行敏感信息检查
- ✅ 检查失败阻止提交
- ✅ 要求用户确认

**AI 无法绕过：**
```bash
# Git Hook 会自动执行
# AI 无法跳过
```

### 提交脚本

**文件：** `scripts/safe_commit.sh`

**作用：**
- ✅ 封装完整流程
- ✅ 强制检查 + 确认
- ✅ AI 必须使用

**用法：**
```bash
./scripts/safe_commit.sh "提交信息"
```
