# 📋 提交规则（强制执行）

**版本：** 1.0  
**生效日期：** 2026-03-10  
**适用范围：** 所有会话、所有提交

---

## ⚠️ 提交前必须完成

### 1. 敏感信息检查 ✅

```bash
uv run scripts/check_secrets.py --all
```

**检查内容：**
- ❌ API Keys（阿里云、GitHub、AWS 等）
- ❌ 飞书配置（AppID、AppSecret、用户 ID）
- ❌ 密码/密钥
- ❌ 数据库连接字符串
- ❌ 私钥文件

**通过标准：**
```
✅ 未发现敏感信息
```

---

### 2. 仓库名确认 ✅

**新 Skill 发布：**
- ✅ 必须使用独立仓库
- ✅ 仓库名必须经用户确认
- ❌ 禁止放在现有仓库中

**确认流程：**
```
AI: "新 Skill 要发布到哪个仓库？"
用户："仓库名：xxx-skill"
AI: "确认发布到 liuqun-a/xxx-skill ？"
用户："确认"
→ 才能执行推送
```

---

### 3. 提交内容确认 ✅

**提交前必须显示：**
```bash
git status --short
```

**用户确认：**
```
AI: "准备提交以下文件，确认吗？"
    [文件列表]
用户："确认"
→ 才能执行 git commit
```

---

### 4. 提交信息规范 ✅

**格式：**
```
<类型>: <描述>

[可选的详细说明]
```

**类型：**
- `feat`: 新功能
- `fix`: 修复 bug
- `docs`: 文档更新
- `style`: 代码格式
- `refactor`: 重构
- `test`: 测试
- `chore`: 构建/工具

**示例：**
```
fix: 修复 ETF 价格处理错误

- ETF 价格除以 1000（不是 100）
- 添加证券类型判断逻辑
```

## ❌ 禁止行为

### 绝对禁止

- ❌ 跳过敏感信息检查
- ❌ 擅自决定仓库名
- ❌ 未确认就提交
- ❌ 新 Skill 放在现有仓库中
- ❌ 硬编码敏感信息

### 不推荐

- ⚠️ 提交大文件（>10MB）
- ⚠️ 提交二进制文件
- ⚠️ 提交 `.gitignore` 中的文件

## 🛡️ 技术约束

### Git Hook（强制）

**文件：** `hooks/pre-commit`

**功能：**
- ✅ 自动执行敏感信息检查
- ✅ 检查失败阻止提交
- ✅ 要求用户确认

**安装：**
```bash
cp hooks/pre-commit .git/hooks/pre-commit
chmod +x .git/hooks/pre-commit
```

### 安全提交脚本（推荐）

**文件：** `scripts/safe_commit.sh`

**用法：**
```bash
./scripts/safe_commit.sh "提交信息"
```

**功能：**
- ✅ 自动检查敏感信息
- ✅ 显示提交内容
- ✅ 强制确认
- ✅ 询问是否推送

**处理：**
